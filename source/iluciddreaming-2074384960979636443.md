# 融合提炼

原 skill 重点来自 Rachel (@Zesee) 的文章：把视频日更从剪辑任务变成可维护的 Remotion 生产线。

本来源补充的重点是：不要只把 Remotion 当成唯一入口。对普通创作者来说，很多赛道一开始并不知道哪种开场、节奏、画面结构和信息密度能跑通，因此更适合先用 HyperFrames 做样片验证，再把稳定格式沉淀到 Remotion。

融合后的原则：

- Codex：负责把内容想法、脚本、规则、组件、检查清单和复盘结论写进项目。
- HyperFrames：负责快速验证新栏目、新画面风格和新内容赛道。
- Remotion：负责长期稳定生产，用 React 组件和 JSON 数据批量渲染。

# 新增模块

本次融合给 skill 增加了四个能力：

1. **引擎选择**：先判断用 HyperFrames 验证、用 Remotion 量产，还是两者结合。
2. **内容赛道分类**：把讲书号、带货讲解、数据科普纳入高复用视频赛道。
3. **样片到模板**：把跑通的 HyperFrames brief 转成 Remotion composition、content JSON 和组件结构。
4. **低成本验证**：新栏目先做 1-3 条样片 brief，再决定是否维护完整模板。
