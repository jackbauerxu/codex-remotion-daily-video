# 来源摘录

这篇 X Article 的核心观点：不要把自媒体视频当成一次性剪辑项目，而要把它当成可维护的代码项目。Codex 负责把想法、脚本、修改意见转成可运行、可复用、可检查的代码和数据；Remotion 负责把 React 组件、JSON 数据、字幕、图片、音频渲染成视频。

关键方法：

- 人负责观点和判断，Codex 负责搭生产线，Remotion 负责稳定出片。
- 适合知识卡片视频、AI 工具教程、数据可视化视频、新闻解读、观点短视频、播客切片包装、产品更新视频、课程切片等结构高度相似的内容。
- 维护一个模板，然后每天喂新数据，是日更痛苦程度的分水岭。
- 在项目根目录写 `AGENTS.md`，把账号规则、视频规格、字幕安全区、标题长度、动画风格、检查方式、导出目录写清楚。
- 最小生产线包括 Remotion 项目、components、theme、content JSON、public assets、render scripts、outputs。
- 每天真正变化的是 content JSON，而不是视频组件。
- 一条视频可以拆成 5 个可复用模块：开场钩子、主体场景、字幕、素材、结尾。
- 每日流程：写原始想法；让 Codex 转成脚本和结构化数据；让 Codex 检查模板是否支持字段；渲染 still 检查；渲染完整视频；复盘并每天修一个小地方。
- 先做三类模板：观点解释型、工具教程型、金句卡片型。
- 真正的无痛来自把规则写进项目，而不是每天在聊天框重复描述需求。
